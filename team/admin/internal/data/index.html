Guidelines

You came up with a nice idea and would like to validate it as a Proof-of-Concept (PoC), then you should validate it first on one of the benchmarking datasets:

  • FedML: A Research Library and Benchmark for Federated Machine Learning
  • LEAF: Benchmark for FL
  • Benchmarking SSFL
  • Real World Image Benchmark FL
  • Benchmarking FL on Clinical Datasets

Once you have a valid PoC, then you could investigate one of the below use-cases. If you are interested to create your own use-case, then please do the following:

  1. Find the database that meets your requirements
  2. Get Shadi’s signature on the corresponding forms, if necessary.
  3. Download the dataset, save it in a secure folder, and add it to the list below
  4. Write the data loader. Please follow the data split (Train/Val/Test) or K-fold CV that appeared in the STOA papers. Please add whatever available meta-information, e.g. Age, Gender, Scanner, Location, …etc.
  5. Store the signed forms, datasets, readme, dataloader, and the Hdf5 in this folder TBD.
  6. Once you design your use-case, append it to the table below.
