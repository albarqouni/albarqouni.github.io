Coding

Guidelines

You came up with a nice idea and would like to code a Proof-of-Concept (PoC), please follow this guidelines:

  1. Create a private repository with detailed readme.md file at Albarqouni Lab. Git.
  2. Make sure you list all dependencies in requirements.yaml
  3. Write all your configurations, e.g. where you retrieve/store the logs and data, in a single JSON file config.json.
  4. Caution: Avoid hard-coding paths or hyper-parameters. Make sure you have a configuration file for every single experiment.
  5. Try to structure your code similar to the example below; dataloader, mains, trainers/optimizers, logs, and utils.
  6. Make sure you have a well-written documentation for your code.
  7. Provide CLI scripts as well as Python Notebooks, if possible.
  8. In case you want to make the repo. publicly available, get Shadi’s approval first in case he wants to file a patent.
  9. Make sure you acknowledge any code your borrowed, and make it clear in the Disclaimer section.
  10. Add the right license, e.g. CC BY-NC-SA 4.0
  11. Push the paper on arXiv and add it to the citation section.
  12. Make sure you adhere to the ML Code Completeness Checklist before you appned your work to the Codes section.

Here is a good example from our recent work on Unsupervised Anomaly Detection.

  .
  Framework/
  │
  ├── readme.md 
  ├── Notebooks.ipynb - Jupyter notebook to work on Google Colab
  ├── run.py - execute to run in commandline
  ├── config.json - holds configuration where to store and retrieve data and logs
  ├── requirements.yaml - list the dependencies required to run the model
  │
  ├── data_loaders/ - Definition of dataloaders
  │   ├── Brain.py
  │   ├── ChestXray.py
  │   └── ...
  │
  ├── logs/ - default directory for storing tensorboard logs
  │
  ├── mains/ - Main files to train each architecture
  │   ├── main_cnn.py
  │   └── ...
  │
  ├── model/ - Architecture definitions
  │   ├── cnn.py
  │   └── ...
  │
  ├── trainers/ - trainers including definition of loss functions, and metrics
  │   ├── cnn.py
  │   └── ...
  │  
  └── utils/ - utility functions
      ├── util.py
      └── ...
